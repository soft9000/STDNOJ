A Frequently Asked Questions

Getting started with any new framework can be daunting. Here are answers to the questions that we see a lot.

General FAQ

FAQ GEN.001: What is the purpose of this project?

 

The purpose of this project is to create and share neat classes and interesting (if not downright useful) tools and applications everywhere C++ is available.

FAQ GEN.002: Where can I get the source code for the project?

 

The project can be found at Sourceforge.net and Freshmeat.net. While from time to time we release a ZIP snapshot of the entire project, the CVS Repository at Sourceforge is where most go to get the latest and greatest.

FAQ GEN.003: How do I choose a platform?

 

Before you can compile, you will need to globally #define your platform as either WIN32, UNIX, DOS, or OSX.

While support for WIN32 enjoys the most comprehensive, support for DOS, UNIX, and Linux are not too far behind. Better support for OSX and others is in the works.

FAQ GEN.004: Is there a tutorial?

 

While the introduction has a lot of background information, those who are eager to see the project in action should probably begin with the tutorial. That document is located in the Project Documentation Folder.

FAQ GEN.005: Are there any examples?

 

You 'betcha - and more are coming available all the time! From 'dweeby little command line tools to full blown graphical applications using Borland C++, Microsoft Visual Studio, .Net, GCC, DJGPP, Qt, Carbon, and DevCPP, we have a lot to share already, as we as a lot to upload yet. Look under the stdnoj/projects folder to see what had been uploaded so far, as well as what else is going on.

FAQ GEN.006: Who uses this project?

 

All told, the project gets between 250 - 600 downloads a month... We estimate that this translates into about 5,000 project investigators a year.

Almost every class and framework has been time-tested in the products available at Soft9000.com, Cookpedia.com, and TheQuoteForToday.com. Please contact us if you would like to have your project added to this list.

FAQ GEN.007: Is there a project 'Blog?

 

From time to time we post some machinations at Blogspace.

FAQ GEN.008: How can we contact you?

 

All of the store-and-forward resources mentioned in this FAQ have varying degrees of messaging support. Alternatively, you can also contact us via our web site. Please note that in order to send an email message to us that before your message can be delivered you will need to provide a valid email address.

(Sorry for the hassle, but after years of email abuse, our new SPAM avoidance strategy requires that we give you our contact email only if you give us yours. Even then, we also change that firewalled email address regularly. (We are even considering sharing a NOJ that helps us manage that, but it has not been perfected yet.))

 

Building FAQ

 

Common questions pertaining to project setup, configuration, and building.

FAQ BLD.001: What do I need to do to create a library on [my_operating_system]?

 

In order to build a library for this project on your platform, all you need is 4 things:

 

(1) Access to the Standard C Library.

(2) Your platform identifier globally #defined (see GEN.003)

(3) The path to stdnoj defined so that #include <stdnoj.hpp> can be readily located

(4) Everything in stdnoj/noj included.

Special Note to Visual Studio Users: It is best NOT to use pre-compiled headers. If you see that stdafx.h garbage included when building your library, then you did something wrong. Best to start off with an empty project (& do not feel bad - I still need to do this every time I build the project, too.) Better still, just open one of the projects under _SUPPORT. Should upgrade nicely.
